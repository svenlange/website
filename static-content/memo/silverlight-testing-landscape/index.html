Silverlight testing landscape

I did some research to find out how to test Silverlight 3 applications. I wanted to have an automated build on a CI server (TeamCity) that does some unit and integration testing plus a code coverage report. Soon I saw that there are a lot of tools and frameworks out there. To get an overview of whats available and useful, I started the following list. Feel free to share your thoughts.

Testing Frameworks

  • Silverlight Unit Test Framework (SLUT) included in the Silverlight Toolkit
    • Code Coverage Howto
    • Tech Ed 2010 - Unit Testing in Microsoft Silverlight (Video)
    • Jeff Wilcox - Blog of the developer behind the framework
  • NUnit
  • MBUnit
  • xUnit.net - Unit Testing for .NET
  • CThru and SilverUnit
    • Unit Testing Silverlight with SilverUnit (Video)
  • TestApi - a library of Test APIs
  • dotCover – .NET Code Coverage Tool
  • NCover - Code Coverage for .NET Developers
  • White - Framework for automating rich client applications based on many different platforms
    • Seems not very mature
    • Silverlight 3 not offically supported yet as of 31 Oct 2010
  • StatLight - Silverlight Testing Automation Tool
  • Telerik - Automated Testing Tools for Web Application Testing (Webcast)
    • WebUI Test Studio QA Edition – $2,499 USD
    • WebUI Test Studio Developer Edition – $1,499 USD
    • WebUI Test Studio Bundle – $2,999 USD
  • Telerik free WebAii Testing Framework
  • silverlight-selenium
    • Too old

IDE Extensions

  • ReSharper - Visual Studio productivity tool
    • AgUnit - Silverlight unit testing with ReSharper
  • dotTrace - Performance and Memory Profiler
  • TestDriven.Net - Zero Friction Unit Testing Add-In
  • Telerik JustCode - Plug-in for Real Time Code Analysis
  • Typemock Unit Testing Tools - Isolator 2010 (Test any code, ..., Increase coverage)
  • Red Gate .NET Developer Bundle
    • ANTS Performance Profiler Pro - Identify bottlenecks and ensure code is performing optimally
    • ANTS Memory Profiler - Find and fix memory leaks in your .NET applications
    • Exception Hunter - Locate the unhandled exceptions your method can throw
    • .NET Reflector Pro - Seamlessly debug into third-party code and assemblies, even if you don't have the source code for them

Mocking

An interesting page about mock frameworks can be found here. One guy there is saying moq has a much cleaner API than Rhino by similar capabilities. Also the guy behind SLUT recommends moq.

  • JustMock Telerik
  • moq - Mocking library for .NET 3.5 and Silverlight with deep C# 3.0 integration
  • Rhino Mocks - A dynamic mock object framework for the .Net platform
  • Pex and Moles - Isolation and White box Unit Testing for .NET
    • Moles - Isolation framework for .NET
